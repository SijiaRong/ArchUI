## 概述

记忆节点是项目知识的持久化层。它不是任务列表，而是对"为什么做出这个决定"的结构化记录，保证在展示材料迭代过程中核心叙事保持一致。

## 节点类型

### 决策节点
记录关键的技术或产品决策及其理由。

格式：
```
- 决策：[做了什么]
- 背景：[当时面临的约束]
- 理由：[为什么这样选]
- 影响：[哪些模块依赖这个决策]
```

### 洞察节点
记录对问题或市场的核心发现。

格式：
```
- 洞察：[发现了什么]
- 来源：[用户访谈 / 数据 / 竞品研究]
- 用途：[在哪些材料中引用]
```

### 叙事锚点
全套材料必须保持一致的核心表述（一句话描述、核心价值主张等）。

## 使用规则

- 新增节点时更新本文件
- 展示材料中的关键表述必须引用此处定义的叙事锚点，不得自行发挥
